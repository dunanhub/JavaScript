## Выбранная функция
Для анализа была выбрана вспомогательная функция `safeFetchJson(url, options)`.

## Какую проблему она решает?
При использование стандарной функции `fetch` часто приходиться повторять один и тот же код:
- Проверять статус ответа (`response.ok`)
- Обрабатывать ошибки
- Парсить JSON
- Возвращать данные в едином формате

Это проводит к дублированию кода и разному поведению при ошибках.

## В каком контексте используется?
- Получение данных с REST API
- Работа с серверными запросами (frontend или Node.js)
- Страницы со списками, дашборды, формы
- Проекты, где много HTTP - запросов

## Как функция решает проблему?
Функция `safeFetchJson` обрачивает `fetch` и выполняет стандартные шаги:
1. Отправляет HTTP - запрос
2. Проверяет статус ответа
3. Безопасно парсит JSON
4. Возвращает данные или понятную ошибку

## Ключевые компоненты решения
- `try/catch` для обработки ошибок
- Проверка HTTP - статуса
- Безопасный JSON - парсинг
- Единый формат возврата данных
