## Название паттерна
Safe Fetch Wrapper Pattern

## Описание
Переиспользуемый паттерн, который стандартизирует работу с HTTP - запросами, оборачивая функцию `fetch` в единый безопасный интерфейс.

## Контекст использования
Паттерн используется, когда:
- Приложение часто обращается к API
- Важно единообразие обработки ошибок
- Над проектом работает несколько разработчиков
- Требуется читаемый и поддерживаемый код

Это усложняет отладку и поддержку приложения.

## Решение
1. Создать функцию - обертку `safeFetchJson`
2. Внутри вызвать `fetch`
3. Проверить HTTP - статус
4. Безопасно распарсить JSON
5. Вернуть данные в стандартном формате

## Дизайн
- Паттерн скрыт от конечного пользователя
- Все логика надежности сосредоточена в одном месте
- Компоненты и сервисы используют единый интерфейс

## Реализация
- Функция размещается в общей папке `utlis`
- Используется ES6 - синтаксис
- Поддерживает гибкие параметры (`url`, `options`)
- Возвращет либо данные, либо понятную ошибку

## Примеры
Минимальный пример реализации приведен в файле `pattern_example.js`.

## Последствия
### Преимущества
- Меньше дублирования кода
- Единая логика обработки ошибок
- Более чистый код компоненты
- Упрощенная отладка

### Недостатки
- Доп. уровень абстракции
- Ошибка в обвертке влияет на все запросы
